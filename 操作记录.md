# 电影评分系统操作记录

项目路径：`D:\Course Project\movie-rating-system`

## 2026-06-30

1. 阅读课程资料中的四份 Word 文档，并将正文抽取为 UTF-8 文本，保存到 `D:\Course Project\word_text_utf8`。
2. 确定选题为“选题一：电影评分系统”。
3. 根据课程要求确定系统范围：
   - 维护电影、演员、导演、用户基本信息。
   - 每部电影关联演员和导演。
   - 用户可以评分、发表评论，并维护自己的评分和评论。
   - 用户评分后，电影平均评分自动变化。
   - 支持按发行年份、类型、评分等条件组合查询和排序。
   - 支持按演员、导演查询。
   - 支持热门电影推荐。
   - 页面提供电影图片和演员信息展示。
4. 检查本机运行环境：
   - Node.js 可用。
   - MySQL 命令行可用。
   - Maven 不可用。
5. 为避免联网下载依赖，采用原生 Node.js 标准库实现本地 Web 系统；数据库部分提供完整 MySQL 脚本，系统演示数据使用本地 JSON 文件持久化。
6. 创建项目目录结构：
   - `public`：前端页面、样式和脚本。
   - `data`：本地 JSON 数据文件。
   - `sql`：MySQL 建库、建表、视图、存储过程、触发器和测试数据脚本。
   - `docs`：课程设计文档素材、数据库设计说明和答辩说明。
   - `reports`：课程设计报告成品。
   - `scripts`：辅助脚本。
7. 生成 `sql/01_schema.sql`：
   - 创建 `movie_rating_system` 数据库。
   - 创建用户、电影、演员、导演、电影演员关联、电影导演关联、评分、评论等表。
   - 设置主键、外键、唯一约束、非空约束、默认值和 CHECK 约束。
   - 创建组合查询需要的索引。
   - 创建 `v_movie_summary` 和 `v_hot_movies` 视图。
   - 创建 `sp_movie_comments`、`sp_actor_movies`、`sp_genre_report` 三个存储过程。
   - 创建评分 INSERT、UPDATE、DELETE 后自动更新电影平均评分和评分人数的触发器。
8. 生成 `sql/02_seed.sql`：
   - 插入管理员、普通用户、电影、演员、导演、关联关系、评分和评论测试数据。
   - 在脚本末尾调用存储过程，便于截图和报告记录测试结果。
