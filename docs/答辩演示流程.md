# 电影评分系统答辩演示流程

## 一、启动系统

在项目目录运行：

```powershell
cd "D:\Course Project\movie-rating-system"
node scripts\start-local.js
```

浏览器访问：

```text
http://127.0.0.1:3000
```

默认管理员账号：

```text
用户名：admin
密码：admin123
```

默认普通用户账号：

```text
用户名：user1
密码：user123
```

## 二、推荐演示顺序

1. 登录系统
   - 使用管理员账号 `admin / admin123` 登录。
   - 展示首页统计：电影数量、演员数量、导演数量、用户数量、评分数量、评论数量。

2. 展示电影列表与组合查询
   - 进入“电影”页面。
   - 按类型筛选，如选择“剧情”。
   - 输入演员姓名，如“张译”。
   - 输入最低评分，如“8”。
   - 切换排序方式，如“评分降序”“热门优先”。

3. 展示电影详情
   - 点击电影“详情”。
   - 展示电影基本信息、演员、导演、评分和评论。

4. 展示评分功能
   - 在电影详情页输入评分，例如 `8.8`。
   - 提交后说明：系统会重新计算电影平均评分和评分人数。
   - 对应数据库脚本中的触发器为：
     - `trg_ratings_after_insert`
     - `trg_ratings_after_update`
     - `trg_ratings_after_delete`

5. 展示评论功能
   - 在电影详情页输入评论并提交。
   - 展示评论列表刷新。

6. 展示基础数据维护
   - 进入“演员”页面，展示演员信息。
   - 进入“导演”页面，展示导演信息。
   - 管理员可新增、编辑、删除未被引用的数据。

7. 展示用户管理
   - 进入“用户”页面。
   - 展示用户角色、状态和邮箱。
   - 说明普通用户没有用户管理权限。

8. 展示报表统计
   - 进入“统计”页面。
   - 展示热门电影推荐、类型评分汇总和用户活跃统计。

9. 展示存储过程对应功能
   - 在“统计”页面执行“电影名查询评论”。
   - 输入“城市”，展示对应电影评论。
   - 执行“演员名查询电影”。
   - 输入“张译”，展示该演员参演电影。
   - 对应数据库脚本中的存储过程为：
     - `sp_movie_comments`
     - `sp_actor_movies`
     - `sp_genre_report`

## 三、数据库重点说明

完整 MySQL 脚本位于：

```text
sql/01_schema.sql
sql/02_seed.sql
```

答辩时可重点说明：

1. 所有基础表都设置了主键。
2. 电影演员、电影导演、评分、评论等表均设置了外键。
3. 电影发行年份、时长、评分、评分人数等字段设置了 CHECK 约束。
4. 用户名唯一，同一用户对同一电影只能有一条评分。
5. 创建了电影综合视图和热门电影视图。
6. 创建了按电影查评论、按演员查电影、按类型统计的存储过程。
7. 创建了评分变更后自动更新电影评分的触发器。

## 四、停止系统

```powershell
node scripts\stop-local.js
```
